음성 읽기 (TTS)

캐릭터 대사 읽기는 별도 서버 없이 OS 내장 음성 합성 (Web Speech API speechSynthesis)을 사용합니다. macOS · Windows 모두 동일 코드 경로로 동작합니다. 앱이 언어(ko/en)에 맞는 로컬 음성을 자동 선택합니다.

← 다운로드 · 로컬 음성 인식(Whisper)

지원 요약

플랫폼엔진비고
macOS AVSpeech / 시스템 음성 (WKWebView) 시스템 설정에 한국어·영어 음성 설치 권장
Windows SAPI / OneCore (WebView2) 설정 → 시간 및 언어 → 음성에서 언어 팩·음성 추가

설정에서 음성 이름을 비워 두면 OS 기본 음성을 사용합니다. 대화 답변과 자발 대사(ambient) 모두 같은 TTS 경로입니다.

앱에서 켜기

  1. 설정 → 행동 탭
  2. 음성 읽기(TTS) 체크
  3. 속도(rate) 조절 (0.5–2.0)
  4. (선택) 한국어/영어 음성 이름을 OS에 표시되는 이름과 동일하게 입력
  5. 저장 후 대화 또는 자발 대사가 소리로 나오는지 확인

언어를 바꾼 뒤 해당 언어 음성이 없으면 안내 토스트가 뜹니다. 그 경우 아래 OS 설정을 확인하세요.

macOS 음성 설치

  1. 시스템 설정 → 손쉬운 사용 → 읽기 도구(Spoken Content) 또는 음성 콘텐츠
  2. 시스템 음성 관리에서 한국어(예: Yuna, Sora) · 영어 음성 다운로드
  3. 또는 시스템 설정 → 일반 → 언어 및 지역 에서 한국어 추가
  4. 앱을 완전히 종료했다가 다시 실행한 뒤 TTS 테스트

Windows 음성 설치

  1. 설정 → 시간 및 언어 → 언어 및 지역 → 한국어/영어 언어 팩 설치
  2. 설정 → 시간 및 언어 → 음성 → 음성 추가
  3. WebView2 Runtime 확인 (Windows 11 기본 포함, 없으면 Evergreen 설치)
  4. 앱 설정에서 TTS 켠 뒤 대화 테스트

음성이 비어 있으면 말풍선은 보이지만 소리가 나지 않을 수 있습니다. 언어 팩 설치 후 앱을 완전히 종료했다가 다시 실행하세요. 시스템 볼륨·앱이 다른 출력 장치를 쓰는지 확인합니다.

STT 와 차이

  • TTS(읽기): OS 내장, 설치 부담 적음, 네트워크 불필요, Win/Mac 기본 지원
  • STT(말하기→글자): 로컬 Whisper 서버 별도 실행 필요 → 설치 가이드
